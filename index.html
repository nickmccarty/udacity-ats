The SQLite Search Cache:
Eliminating Redundant API Calls
in Multi-Session Research

When autoresearch runs across multiple sessions, it re-issues the same web search queries on every loop — burning API credits, hitting rate limits, and adding 30–90 seconds of latency per session with identical round-trips. This lesson teaches the production fix: a SHA-256-keyed SQLite cache with configurable TTL that intercepts queries before they reach the API, stores results on miss, and serves them from disk on hit.

Learning Objective

By the end of this lesson, you should be able to implement a cached_search() wrapper backed by a SQLite database that normalises query strings to cache keys via SHA-256, stores results with configurable TTL, serves cache hits without calling the API, and evicts expired entries on every write — reducing multi-session API call volume to zero for repeated queries within the TTL window.

Key Concepts

How the Cache Works

Query Normalisation Before Hashing

SHA-256 is case-sensitive and whitespace-sensitive. "Flash Attention" and "flash attention" produce completely different digests. The normalisation step collapses these variants to a canonical form before hashing:

def _cache_key(query: str) -> str:
    normalised = " ".join(query.lower().split())
    return hashlib.sha256(normalised.encode()).hexdigest()

" ".join(query.lower().split()) lowercases the string and collapses all whitespace sequences — tabs, multiple spaces, newlines — to a single space. Only then is SHA-256 applied.

The Cache Table Schema

CREATE TABLE IF NOT EXISTS search_cache (
    key        TEXT PRIMARY KEY,
    query      TEXT NOT NULL,
    results    TEXT NOT NULL,
    created_at REAL NOT NULL DEFAULT 0,
    expires_at REAL NOT NULL DEFAULT 0
);
CREATE INDEX IF NOT EXISTS idx_expires ON search_cache(expires_at);

The SHA-256 hex digest is the primary key. results stores the API response as a JSON string. expires_at is a Unix timestamp; the index makes the eviction scan fast at scale.

Get / Put / Eviction

get() — read path
  • Normalise query → compute key
  • Look up row by key
  • If row is None: return None (MISS)
  • If expires_at < time.time(): delete row, return None (expired)
  • Otherwise: return json.loads(results)
put() — write path
  • Normalise query → compute key
  • Upsert with ON CONFLICT(key) DO UPDATE
  • Set expires_at = time.time() + ttl
  • Inline eviction: DELETE WHERE expires_at < now
  • No background timer needed
Why inline eviction? Deleting stale rows on every write keeps the database from growing unboundedly without requiring a background thread or scheduled job. The index on expires_at makes this scan cheap even at tens of thousands of rows.

The Upsert Pattern

INSERT INTO search_cache (key, query, results, created_at, expires_at)
VALUES (?, ?, ?, ?, ?)
ON CONFLICT(key) DO UPDATE SET
    results    = excluded.results,
    created_at = excluded.created_at,
    expires_at = excluded.expires_at

If the same key is inserted twice (e.g. the cache was cleared and the query re-run within one session), the upsert silently updates the existing row instead of erroring. This is essential for idempotent multi-session behaviour.

The cached_search() Wrapper

def cached_search(
    query: str,
    search_fn: Callable[[str, int], list[dict]],
    ttl: int = DEFAULT_TTL,
    max_results: int = 10,
) -> list[dict]:
    cached = get(query)
    if cached is not None:
        print(f"[cache HIT ] {query[:60]}")
        return cached
    print(f"[cache MISS] {query[:60]}")
    results = search_fn(query, max_results)
    if results:
        put(query, results, ttl=ttl)
    return results

search_fn is any callable matching the (str, int) -> list[dict] signature — DDGS, Tavily, Bing, or a test stub. The wrapper is search-provider-agnostic.

Curriculum Gap Justification

Why This Lesson Exists

Course 4 scope reference: S-C4.3 teaches web search integration, query engineering, and rate limiting defence. S-C4.8 covers multi-source search orchestration and search budgeting. Neither addresses query-level result persistence across sessions.

In single-session exercises, search result reuse is invisible — each session starts fresh and the cost of redundant queries is confined to that session. In multi-session workflows like autoresearch, the cost is multiplicative: a 12-query session run 10 times produces 120 API calls, of which 108 are typically redundant.

The production harness addresses this with two cache tables: search_cache (per-query results, always active) and research_cache (full gather_research() output, opt-in via RESEARCH_CACHE=1). The second table reduces autoresearch session startup from 3–8 minutes of search round-trips to a single SQLite lookup — a latency reduction that compounds across every subsequent session on the same topic.

Connection to S-C4.6 Memory Search: The same caching architecture scales from individual queries to full research contexts. Session memory and search caching are the same architectural pattern at different granularities.

Production Workflow

Each artifact in this lesson — slides, video, PPTX deck — is generated programmatically. No manual slide editors, no screen-recording software, no post-production timeline.

01
Content Design
Lesson plan, Bloom’s alignment, and recording script authored in Markdown. Segment boundaries mapped to natural sentence pauses identified in transcripts of the voice recordings.
02
HTML Slide Generation — produce_video.py
11 animated HTML slides are generated in memory at 1280×720, styled with Udacity design tokens (Open Sans, Roboto Mono, navy #171A53, seafoam #00C5A1, brand blue #2015FF). Each element fades in on a precise millisecond schedule via a lightweight JavaScript event player.
python produce_video.py  →  slides/slide01.html … slide11.html
03
Headless Screen Recording — Playwright
A local HTTP server serves the slide directory. Playwright (headless Chromium, device_scale_factor=1.5 for crisp sub-pixel rendering) navigates to each slide, waits 2.5 s for Google Fonts to settle, fires the animation event scheduler, and records the result as a .webm.
sync_playwright → browser.new_context(record_video_dir=…) → seg01.webm …
04
Audio Clip Extraction — ffmpeg
Each segment maps to a sub-clip of one of six source WAV files recorded in one take. Clip boundaries were chosen at silence gaps detected with ffmpeg silencedetect -35dB and verified against timestamped transcripts. ffmpeg extracts each clip with sample-accurate -ss / -t parameters.
ffmpeg -ss {ss} -t {dur} -i source.wav seg01_clip.wav
05
Merge & Concatenate — ffmpeg
Each .webm + .wav pair is merged into an H.264 MP4 at CRF 15 using -shortest to trim to audio length. All 11 segments are concatenated with the concat demuxer (stream-copy, no re-encode) to produce the final MP4.
ffmpeg -f concat -safe 0 -i concat_list.txt -c copy sqlite-search-cache-lesson.mp4
06
PPTX Deck — build_deck.py
A parallel script generates a 20″×11.25″ PPTX deck aligned to the Udacity teaching-sample template — same 11 slides, same Udacity light theme tokens, speaker notes matching the recording script — using python-pptx programmatically.
python build_deck.py  →  sqlite-search-cache.pptx
07
Deployment — Cloudinary
Final MP4 uploaded to Cloudinary and embedded via the Cloudinary Player iframe with responsive aspect-ratio:16/9 sizing. No self-hosted video infrastructure required.

def record_segment(browser, slide_num, url, duration_s):
    """Record one slide via Playwright headless Chromium."""
    ctx = browser.new_context(
        viewport={"width": W, "height": H},
        device_scale_factor=1.5,          # crisp sub-pixel rendering
        record_video_dir=str(SEGMENTS_DIR),
        record_video_size={"width": W, "height": H},
    )
    page = ctx.new_page()
    page.goto(url, wait_until="load")
    time.sleep(2.5)                        # wait for Google Fonts
    page.evaluate("play(window.__EVENTS__)")
    time.sleep(duration_s)
    ctx.close()
    recorded = Path(page.video.path())
    shutil.move(str(recorded), SEGMENTS_DIR / f"seg{slide_num:02d}.webm")

def merge_av(webm, wav, out):
    """Merge screen recording + audio clip into a single .mp4 segment."""
    ff("-i", str(webm), "-i", str(wav),
       "-c:v", "libx264", "-pix_fmt", "yuv420p",
       "-preset", "fast", "-crf", "15",
       "-c:a", "aac", "-ar", "48000", "-ac", "2",
       "-shortest", "-movflags", "+faststart", str(out))

def concat_segments(mp4_files, final):
    """Stream-copy all segment MP4s into the final video (no re-encode)."""
    concat_list = SEGMENTS_DIR / "concat_list.txt"
    concat_list.write_text(
        "\n".join(f"file '{p}'" for p in mp4_files), encoding="utf-8"
    )
    ff("-f", "concat", "-safe", "0", "-i", str(concat_list),
       "-c", "copy", str(final))
